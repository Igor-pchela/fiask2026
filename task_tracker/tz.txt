Техническое задание на разработку веб-приложения «Трекер задач»

1. Введение
1.1. Описание проекта
Веб-приложение «Трекер задач» представляет собой систему для постановки, отслеживания и управления задачами в рамках командной или индивидуальной работы. Пользователи могут создавать задачи, назначать приоритеты, устанавливать дедлайны, оставлять комментарии с файлами, изменять статусы и просматривать статистику выполнения. Приложение обеспечивает разграничение прав (администратор, обычный пользователь), поддержку публичных и личных задач, фильтрацию, поиск и удобный интерфейс для повседневного использования.
1.2. Цели и задачи
•	Создать работоспособное веб-приложение на Flask, соответствующее современным стандартам веб-разработки.
•	Реализовать полный цикл управления задачами: создание, редактирование, изменение статуса, комментирование, архивация.
•	Обеспечить безопасную аутентификацию и авторизацию пользователей.
•	Предоставить удобные инструменты фильтрации, сортировки и поиска задач.
•	Реализовать статистику выполнения задач за период (неделя, месяц).
•	Гарантировать корректную работу на всех современных браузерах и устройствах.
1.3. Целевая аудитория
•	Руководители проектов, тимлиды, менеджеры – для контроля выполнения задач командой.
•	Разработчики, дизайнеры, тестировщики – для ведения личных и рабочих задач.
•	Фрилансеры и индивидуальные пользователи – для планирования личных дел.
•	Администраторы – для управления системой и пользователями.

2. Требования к функциональности
2.1. Основные функции сайта
•	Управление пользователями: регистрация, аутентификация по логину/паролю, разграничение ролей (admin, user).
•	Управление задачами: создание, просмотр, редактирование, удаление (только для администратора/автора), изменение статуса, отправка в архив.
•	Комментирование: возможность добавлять текстовые комментарии к любой задаче с прикреплением файлов.
•	Фильтрация и поиск: фильтрация по статусу, приоритету, дедлайну (до), дате создания (от); поиск по названию и описанию.
•	Сортировка: по статусу (жизненный цикл new → in_progress → done → archived), дате создания, дедлайну, приоритету.
•	Статистика: отображение количества выполненных задач за последние 7 и 30 дней (для текущего пользователя или всех, если админ).
•	Загрузка файлов: к задаче и к комментарию – любые типы файлов (до 16 МБ).
2.2. Регистрация и авторизация пользователей
•	Форма регистрации: поля «Логин» (уникальный) и «Пароль» (минимум 6 символов).
•	Форма входа: поля «Логин», «Пароль».
•	После успешного входа пользователь перенаправляется на список своих задач.
•	Для неавторизованных пользователей доступ только к страницам входа/регистрации.
•	При попытке доступа к защищённым страницам – редирект на логин.
2.3. Возможность добавления, редактирования и удаления контента
•	Задачи: создание (название, описание, приоритет, дедлайн, видимость, файл). Редактирование – только автором или админом (кроме архивных). Удаление задач не предусмотрено (только архивация).
•	Комментарии: добавление, редактирование и удаление не предусмотрены (только создание).
•	Пользователи: администратор может менять роли или удалять пользователей (опционально, в будущих версиях).
2.4. Поиск и фильтрация данных
•	Поисковая строка в навигации: поиск по названию и описанию задачи среди доступных пользователю задач.
•	Фильтры на странице списка задач: статус (all, new, in_progress, done, archived), приоритет (all, high, medium, low), дедлайн до (дата), создана от (дата).
•	Сортировка: выбор порядка отображения.
2.5. Интеграция с внешними сервисами
•	На данном этапе интеграция не требуется. В перспективе – уведомления по email, импорт/экспорт задач через API или CSV.

3. Дизайн и пользовательский интерфейс
3.1. Общий стиль и цветовая гамма
•	Стиль – минималистичный, светлая тема (основной фон #f8f9fa, карточки белые, тени).
•	Акцентные цвета: синий (основные действия), зелёный (успех, готово), жёлтый (предупреждение), красный (высокий приоритет, ошибка).
•	Шрифты: системные (sans-serif), адаптивная типографика.
3.2. Основные элементы интерфейса
•	Навигационная панель: логотип, ссылки «Задачи», «Создать», «Статистика», строка поиска, имя пользователя с ролью, кнопка «Выйти».
•	Страница задач: переключатель «Мои / Публичные», форма фильтров и сортировки, карточки задач (с заголовком, описанием, метками приоритета и статуса, датами, автором), пагинация.
•	Страница задачи: детали, блок изменения статуса, список комментариев с возможностью добавления комментария и файла, кнопки «Редактировать», «В архив» (если доступно).
•	Формы: создание/редактирование задачи, комментарий – стилизованы с помощью Bootstrap.
3.3. Макеты страниц сайта
•	Главная после входа – страница «Мои задачи».
•	Отдельные страницы: список публичных задач, детальная задача, форма создания/редактирования, статистика, результаты поиска.
3.4. Адаптивный дизайн для различных устройств
•	Использование Bootstrap 5: сетка, адаптивные карточки, сворачивающееся меню на мобильных устройствах.
•	Корректное отображение на экранах шириной от 320px до 1920px.

4. Требования к контенту и структуре
4.1. Типы контента
•	Тексты: название задачи, описание, комментарии.
•	Изображения и документы: любые файлы, прикреплённые к задачам и комментариям.
•	Метки данных: статусы, приоритеты, даты, имена пользователей.
4.2. Структура сайта
•	Главная.
•	Публичные задачи.
•	Создание задачи.
•	Просмотр задачи.
•	Редактирование.
•	Статистика.
•	Поиск.
•	Вход/выход/регистрация.

5. Технические требования
5.1. Технологии разработки
•	Язык: Python 3.10+
•	Фреймворк: Flask 2.3.3
•	ORM: SQLAlchemy 3.1.1
•	База данных: SQLite (разработка), PostgreSQL (продакшн)
•	Аутентификация: Flask-Login
•	Формы: Flask-WTF, WTForms
•	Фронтенд: Bootstrap 5, HTML5, CSS3, JavaScript (минимально)
•	Загрузка файлов: Werkzeug
5.2. Поддержка браузеров и ОС
•	Последние версии: Google Chrome, Mozilla Firefox, Microsoft Edge, Safari.
•	Операционные системы: Windows, macOS, Linux.


6. Требования к поддержке и обслуживанию после запуска
6.1. Техническая поддержка и обслуживание
•	Гарантийное сопровождение в течение 1 месяца после сдачи.
•	Внеплановая поддержка – по договорённости.
•	Мониторинг ошибок через логи Flask.
